LMS Hub

Pathways is the parent of all LMSs. Plug it into Canvas, Moodle, Blackboard, Brightspace, Google Classroom, Schoology, D2L, or any other system that speaks the open standards below. Import a course, export a lesson, ship analytics — all from one place.

Standards Pathways speaks

Every major LMS uses these open standards. When Pathways supports them, it plugs in.

StandardWhat it doesPathways support
LTI 1.3
1EdTech		 Launch Pathways tools from any LMS (Canvas, Moodle, Blackboard, Brightspace, Schoology, Google Classroom, D2L) with SSO. Supports Deep Linking + AGS + NRPS. Configuration ready · server-side launch (self-host)
Common Cartridge 1.3
1EdTech		 Import/export entire course packages (lessons, assessments, resources) as .imscc files between LMSs. Export built-in ↓
Thin Common Cartridge
1EdTech		 Minimal Common Cartridge with just web-link resources — fast course migration. Export built-in ↓
SCORM 1.2 / 2004
ADL		 Packaged self-contained e-learning courses with completion/score tracking. Legacy but still required by many enterprise LMSs. Export built-in ↓
cmi5
ADL		 The modern replacement for SCORM — an xAPI profile with SCORM-style course lifecycle. Export built-in ↓
xAPI (Experience API)
ADL		 Record learning as actor → verb → object statements in a Learning Record Store (LRS). Works offline and across devices. Statement builder ↓
QTI 2.2 / 3.0
1EdTech		 Exchange assessment items and tests between platforms. Export built-in ↓
OneRoster 1.2
1EdTech		 Rostering — sync classes, students, enrolments between an SIS and learning platforms. CSV format ready · API sync requires server
Caliper Analytics
1EdTech		 Event-driven learning analytics with a stricter profile model than xAPI. Documented · add LRS endpoint
OpenBadges 3.0
1EdTech		 Verifiable digital credentials for achievements and completions. Documented · link external issuer
IMS Global / 1EdTech
umbrella		 The consortium maintaining most of the above. Pathways follows their conformance guides. Aligned
Markdown / PDF / JSON Universal fallbacks — anything can read them. Export built-in ↓

LTI 1.3 integration

LTI 1.3 launch requires a server (signed JWT verification). When Pathways is self-hosted, give your LMS admin this registration info:

{ "client_name": "Pathways", "initiate_login_uri": "https://YOUR-DOMAIN/lti/login", "redirect_uris": ["https://YOUR-DOMAIN/lti/launch"], "public_jwks_uri": "https://YOUR-DOMAIN/.well-known/jwks.json", "deployment_id": "provided by LMS admin", "scope": "openid https://purl.imsglobal.org/spec/lti-ags/scope/lineitem https://purl.imsglobal.org/spec/lti-ags/scope/score https://purl.imsglobal.org/spec/lti-nrps/scope/contextmembership.readonly", "services": ["AGS", "NRPS", "DeepLinking"] }

For the static hosted version of Pathways, your LMS can still link out via a standard web link — just paste https://YOUR-DOMAIN/path-builder.html?role=teachers&curriculum=XXX as a course resource.
