Welcome to Pathways

Pathways is an all-in-one platform that brings together the apps, resources, and frameworks needed across every education role — teachers, learners, schools, institutions, parents, and researchers. This guide walks you through how the site is organized and gives you a starting point for your specific path.

Note

You don't need an account to browse Pathways. App submissions and resource contributions are handled through pull requests on our public repo — see Submitting your work.

# How Pathways is organized

Pathways is split into four primary areas, reachable from the top nav:

  • Paths — role-specific entry points for teachers, learners, schools, institutions, parents, and researchers.
  • Apps — focused tools: lesson planners, classroom managers, assessment builders, study aides, administration software, and analytics.
  • Resources — templates, policies, cheatsheets, open courseware, datasets, and reference material curated per role.
  • Docs — guides (this page), a reference for frameworks and standards, an API reference, and an FAQ.

# For teachers

If you're in the classroom, start here:

  • Plan lessons with LessonLoom, build rubrics with RubricRaft, and run formative checks with QuizForge.
  • Use ClassroomCompass for seat maps, timers, random picker, and behaviour tracking.
  • Browse the Teachers resource path for UDL templates, Bloom's stems, Common Core maps, and OER.
  • When adopting a new tool, check it against the SAMR model to avoid "tech for tech's sake."

# For learners

Whether you're in school, university, or self-directed:

  • FlashFlow for spaced-repetition study, MindMapr for concept notes, and StudyBuddy when you need a Socratic partner.
  • HomeworkHero keeps assignments on track; CiteRight handles references.
  • The Learners resource path links to Khan Academy, MIT OCW, OER Commons, and citation guides.

# For schools

K-12 school leaders and operations staff:

  • Timetabler and Attendly for core ops; ParentLine for family communication in 40+ languages.
  • PolicyForge assembles safeguarding, AI-use, and data-protection policies from vetted templates.
  • The Schools resource path covers FERPA, GDPR-K, WCAG 2.2, and safeguarding checklists.

# For institutions

Universities, colleges, and large training organizations:

  • Integrate Pathways apps with your LMS via LTI 1.3; ship courseware as SCORM or xAPI.
  • InsightsLED surfaces at-risk students from xAPI event streams; OpenLecture captions and publishes lecture video.
  • The Institutions resource path collects accreditation toolkits, learning-analytics handbooks, and PISA datasets.

# For parents

Supporting learning at home:

  • HomeworkHero (parent view) and ParentLine for communicating with teachers.
  • The Parents resource path has age-banded reading lists and evidence-based screen-time guidance.

# For researchers

Education research and learning analytics:

  • SignalPath for classroom-safe A/B experiments; CiteRight for reference management.
  • The Researchers resource path links open datasets (PISA, IPEDS) and the SoLAR Handbook of Learning Analytics.

# Submitting your work

Apps and resources live as JSON entries in the repo. To add one, fork the repo, append a new entry, and open a pull request. A typical app entry looks like this:

{
  "id": "lesson-weaver",
  "name": "LessonWeaver",
  "category": "Lesson Planning",
  "description": "Draft differentiated lesson plans from a single learning objective.",
  "tags": ["teachers", "planning", "differentiation"],
  "icon": "🧵",
  "url": "https://lessonweaver.example"
}

Resource entries follow a similar shape but add a type field (template, policy, framework, courseware, dataset, guide, reference, checklist, cheatsheet, tool) and one or more paths.

Review checklist

  • The link works and points directly to the project — not a tracker, paywall, or landing page.
  • At least two relevant tags; a clear category or list of paths.
  • Accessibility: the app is usable with a keyboard and screen reader where feasible.
  • Privacy: if the app handles student data, it discloses how (and ideally is FERPA/GDPR-K aware).
Tip

Run npm run lint:data locally before committing — it catches most schema mistakes in seconds.

# Keyboard shortcuts

  • g then h — go to Home
  • g then p — jump to Paths
  • g then a — go to Apps
  • g then r — go to Resources
  • g then d — go to Docs
  • / — focus the search field on the current page
  • ? — open the shortcut help overlay

Shortcuts are disabled while you're typing into a text field.
