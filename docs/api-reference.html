API reference

The Pathways API is read-only and serves the same data that powers the directory. Base URL:

https://api.pathways.example/v1

# Authentication

Public endpoints require no auth. Institutional endpoints (rostering, xAPI forwarding) use OAuth 2.0 client-credentials; request a client ID at api@pathways.example.

# Apps

GET /apps

List all apps. Supports filtering and pagination.

Query parameters

  • category — Lesson Planning, Assessment, Classroom, Study, Administration, Communication, Accessibility, Analytics
  • path — teachers, learners, schools, institutions, parents, researchers
  • tag — any tag string (repeatable)
  • q — full-text search across name, description, and tags
  • page (default 1), per_page (default 20, max 100)

Example

GET /v1/apps?category=Assessment&path=teachers&per_page=5

Response

{
  "data": [
    {
      "id": "quizforge",
      "name": "QuizForge",
      "category": "Assessment",
      "description": "Create auto-graded quizzes...",
      "tags": ["teachers", "quiz", "formative"],
      "icon": "📝",
      "url": "/apps/quizforge.html",
      "standards": ["qti-3.0", "lti-1.3"]
    }
  ],
  "page": 1,
  "per_page": 5,
  "total": 1
}

GET /apps/{id}

Return a single app by its id. Returns 404 if not found.

# Resources

GET /resources

List resources. Same pagination conventions as /apps.

Query parameters

  • type — template, policy, framework, courseware, dataset, guide, reference, checklist, cheatsheet, tool
  • path — same set as apps
  • license — SPDX identifier (e.g. CC-BY-4.0, MIT)
  • q — full-text search

GET /resources/{id}

Return a single resource by its id.

# Paths

GET /paths

List the six audience paths with counts of apps and resources in each.

    { "id": "parents",      "apps":  4, "resources":  9 },

  ]
}

# Frameworks

GET /frameworks

List frameworks that Pathways indexes (LTI, xAPI, SCORM, QTI, UDL, WCAG, FERPA, etc.) grouped by category: interop, pedagogy, compliance, accessibility, curriculum.

# Errors

The API returns standard HTTP status codes. Errors have a JSON body:

{
  "error": {
    "code": "not_found",
    "message": "No app with id 'lessonweaver'."
  }
}
  • 400 bad_request — invalid query parameters
  • 404 not_found — unknown id
  • 429 rate_limited — more than 60 requests/min per IP
  • 503 service_unavailable — upstream maintenance
Rate limits

Public endpoints: 60 req/min per IP. Authenticated institutional endpoints: 600 req/min per client. Responses include X-RateLimit-Remaining and X-RateLimit-Reset headers.
